一、项目概述
 
TVAiPlatform 是一个集成化 AI 短剧生成平台，整合 Dify（LLM 编排）、ComfyUI（图像/视频生成）及多种 AI 能力，实现从剧本到成片的全流程自动化生产。所有外部服务均支持本地部署或接入第三方云平台，用户可根据资源灵活选择。
 
核心理念：AI 驱动生产 + 人工精细打磨。AI 负责批量生成初稿，人工对文本、图片、人声、背景音、视频等每个环节进行逐项调整与质量把控。
 
1.1 核心价值
 
能力	描述
人物一致性	贯穿全剧的角色外貌、年龄、体型、服装、配饰严格一致，支持青/中/老年跨年龄演绎
多角色同框	多人物交互场景，位置、遮挡、光影合理
音画字同步	人声、背景音、字幕与视频帧精确对齐，情感对话自然

2.2 服务部署方式 — 本地 / 第三方平台灵活选择
 
所有外部服务均支持两种模式：本地部署和接入第三方平台。通过 config.yaml 统一配置，切换只需修改地址与密钥。
 
服务	本地部署	第三方平台	说明
Dify	本地 Docker 容器	Dify Cloud / 自建远程实例	通过 API URL + API Key 接入
ComfyUI	本地进程 (默认 8188)	远程 GPU 服务器 / RunComfy / ThinkDiffusion 等云服务	支持 http/https，自动适配
TTS (ChatTTS)	本地进程 (默认 8090)	第三方 TTS API（如 FishAudio、MiniMax Speech）	通过引擎名切换
TTS (CosyVoice)	本地进程 (默认 8091)	阿里云 CosyVoice API	本地/云端统一接口
TTS (GPT-SoVITS)	本地进程 (默认 8092)	第三方托管实例	本地/云端统一接口
LLM (Ollama)	本地进程 (默认 11434)	OpenAI API / DeepSeek API / 通义千问 API	Dify 后端模型，可替换
PostgreSQL 17	本地进程 (5432)	云数据库（如阿里云 RDS、AWS RDS）	修改连接串即可
Redis 7.4	本地进程 (6379)	云 Redis（如阿里云 Redis、AWS ElastiCache）	修改连接串即可
FFmpeg 7.1	系统安装	—	纯本地工具，无需远程
 
设计原则: 服务调用层对上层业务完全透明。无论是本地 http://127.0.0.1:8188 还是远程 https://comfyui.cloud.example.com，业务代码无需任何修改，仅通过配置切换。

三、人工输入与全局设定体系
 
3.1 设计理念
 
平台中存在大量无法穷举的主观创意输入，如整体动漫风格、画面色调偏好、叙事节奏、角色性格基调等。这些输入不应硬编码在系统中，而应通过全局设定 + 场景级覆盖 + 模板可配的三层机制，让用户自由定义任何维度的创作偏好。

3.3 全局设定数据模型

python
from pydantic import BaseModel, Field
from typing import Dict, Optional


class ProjectGlobalSettings(BaseModel):
    """项目全局设定 — 所有维度均可自定义扩展"""

    # ── 预设维度（用户可从下拉选择或手动输入） ──
    art_style: str = "anime"              # 画风: anime / realistic / watercolor / pixel / 自定义
    color_palette: str = "warm"           # 色调: warm / cool / muted / vibrant / 自定义
    narrative_pace: str = "fast"          # 叙事节奏: fast / moderate / slow / 自定义
    target_audience: str = "young_adult"  # 目标受众
    overall_mood: str = "action"          # 整体氛围
    music_style: str = "electronic_pop"   # 音乐风格
    subtitle_style: str = "white_stroke"  # 字幕风格

    # ── 自定义维度（用户自由添加任意键值对） ──
    custom_dimensions: Dict[str, str] = Field(default_factory=dict)
    # 示例: {"镜头语言": "手持跟拍为主", "对白风格": "口语化、短句", "特效偏好": "粒子光效"}

    # ── LLM 提示词前缀（自动拼接到所有 LLM 请求前） ──
    global_prompt_prefix: str = ""

    def to_prompt_context(self) -> str:
        """将全局设定转换为 LLM 可理解的上下文文本"""
        lines = []
        lines.append(f"画风: {self.art_style}")
        lines.append(f"色调: {self.color_palette}")
        lines.append(f"叙事节奏: {self.narrative_pace}")
        lines.append(f"目标受众: {self.target_audience}")
        lines.append(f"整体氛围: {self.overall_mood}")
        lines.append(f"音乐风格: {self.music_style}")
        for key, value in self.custom_dimensions.items():
            lines.append(f"{key}: {value}")
        if self.global_prompt_prefix:
            lines.append(f"额外要求: {self.global_prompt_prefix}")
        return "\n".join(lines)


class SceneOverride(BaseModel):
    """场景级设定覆盖 — 只覆盖非空字段"""
    scene_id: str
    color_palette: Optional[str] = None
    mood: Optional[str] = None
    extra_prompt: Optional[str] = None
    custom_overrides: Dict[str, str] = Field(default_factory=dict)

四、JSON 模板体系 — 驱动 LLM 结构化输出
 
4.1 设计理念
 
Dify 是使用 LLM 的工具，LLM 需要明确的 JSON 结构模板来输出结构化数据，而不是自然语言描述。系统中所有需要 LLM 输出结构化数据的环节，都通过可编辑的 JSON 模板来驱动。用户可在界面上可视化编辑这些模板，调整输出结构。

4.5 Dify 调用时的模板注入机制
 
Dify 调用 LLM 时，不是用自然语言描述"请输出包含 title、genre 等字段的 JSON"，而是将完整的 JSON 模板作为示例直接注入到提示词中：

python
class DifyClient:
    """Dify API 客户端 — 模板驱动的结构化输出"""

    def __init__(self, base_url: str, api_key: str, template_service: "TemplateService") -> None:
        self.base_url = base_url.rstrip("/")
        self.api_key = api_key
        self.template_service = template_service
        self._client = httpx.AsyncClient(timeout=120.0)

    async def generate_script(self, params: dict, project_settings: "ProjectGlobalSettings") -> dict:
        """调用剧本创作助手 — 注入 JSON 模板 + 全局设定"""
        # 获取模板
        template = await self.template_service.get_template("script_outline")

        # 构建完整提示词：全局设定上下文 + JSON 模板示例
        global_context = project_settings.to_prompt_context()
        template_example = json.dumps(template.example, ensure_ascii=False, indent=2)

        enhanced_params = {
            **params,
            "global_settings": global_context,
            "output_json_template": template_example,
            "system_prompt_suffix": template.system_prompt_suffix,
        }

        response = await self._call_workflow("script_writer", enhanced_params)
        return json.loads(response["script"])

    async def generate_dialogues(self, params: dict, project_settings: "ProjectGlobalSettings") -> list:
        """调用对话生成器 — 注入对话 JSON 模板"""
        template = await self.template_service.get_template("dialogue")
        global_context = project_settings.to_prompt_context()
        template_example = json.dumps(template.example, ensure_ascii=False, indent=2)

        enhanced_params = {
            **params,
            "global_settings": global_context,
            "output_json_template": template_example,
            "system_prompt_suffix": template.system_prompt_suffix,
        }

        response = await self._call_conversation("dialogue_generator", enhanced_params)
        return json.loads(response["answer"])

    async def _call_workflow(self, app_name: str, inputs: dict) -> dict:
        url = f"{self.base_url}/v1/workflows/run"
        headers = {"Authorization": f"Bearer {self.api_key}"}
        payload = {"inputs": inputs, "response_mode": "blocking", "user": "tvai-system"}
        resp = await self._client.post(url, json=payload, headers=headers)
        resp.raise_for_status()
        return resp.json()["data"]["outputs"]

    async def _call_conversation(self, app_name: str, inputs: dict) -> dict:
        url = f"{self.base_url}/v1/chat-messages"
        headers = {"Authorization": f"Bearer {self.api_key}"}
        payload = {"inputs": inputs, "response_mode": "blocking", "user": "tvai-system"}
        resp = await self._client.post(url, json=payload, headers=headers)
        resp.raise_for_status()
        return json.loads(resp.json()["answer"])

五、无限集数支持 — 上下文压缩与分层摘要
 
5.1 问题分析
 
LLM 存在 token 上下文限制。当短剧集数增多（如 50 集、100 集甚至更多），将全部剧本、角色历史、场景信息一次性送入 LLM 会超出 token 限制。需要通过分层摘要 + 上下文压缩 + 滑动窗口机制来突破此限制。
 
5.2 分层摘要架构
 
text
text
┌─────────────────────────────────────────────────────────────┐

│  内容: 整体故事线、核心角色关系、主要冲突、主题               │
│  大小: ≤ 500 tokens                                          │

class ContextCompressor:
    """上下文压缩引擎 — 支持任意集数"""

    MAX_PROJECT_SUMMARY_TOKENS = 500
    MAX_ARC_SUMMARY_TOKENS = 300
    MAX_EPISODE_SUMMARY_TOKENS = 200
    SLIDING_WINDOW_SIZE = 2            # 滑动窗口：当前集 + 前 N 集

    async def build_context_for_episode(
        self,
        project_id: str,
        target_episode_number: int,
        project_settings: "ProjectGlobalSettings",
    ) -> str:
        """为指定集构建 LLM 上下文 — 自动压缩，永不超限"""

        context_parts = []

        # 1. 全局设定（固定开销，约 100-300 tokens）
        context_parts.append("=== 全局设定 ===")
        context_parts.append(project_settings.to_prompt_context())

        # 2. 项目级摘要（固定开销，≤ 500 tokens）
        project_summary = await self.get_project_summary(project_id)
        context_parts.append("\n=== 故事整体概要 ===")
        context_parts.append(project_summary)

        # 3. 相关故事弧摘要（仅当前弧 + 前一弧，约 300-600 tokens）
        relevant_arcs = await self.get_relevant_arcs(project_id, target_episode_number)
        for arc in relevant_arcs:
            context_parts.append(f"\n=== 故事弧: {arc.name} ===")
            context_parts.append(arc.summary)

        # 4. 前几集摘要（滑动窗口，约 200-400 tokens）
        prev_episodes = await self.get_previous_episode_summaries(
            project_id, target_episode_number, window=self.SLIDING_WINDOW_SIZE
        )
        for ep in prev_episodes:
            context_parts.append(f"\n=== 第{ep.number}集摘要: {ep.title} ===")
            context_parts.append(ep.summary)

        # 5. 当前集完整内容（如果存在，用于修改场景）
        current_episode = await self.get_episode_full_content(project_id, target_episode_number)
        if current_episode:
            context_parts.append(f"\n=== 第{target_episode_number}集完整内容 ===")
            context_parts.append(current_episode)

        # 6. 角色当前状态（非全部历史，仅当前状态快照）
        character_states = await self.get_character_current_states(project_id, target_episode_number)
        context_parts.append("\n=== 角色当前状态 ===")
        context_parts.append(character_states)

        return "\n".join(context_parts)

    async def generate_episode_summary(self, episode_full_content: dict) -> str:
        """自动生成集级摘要 — 每集完成后调用"""
        # 使用 LLM 将完整集内容压缩为 ≤ 200 tokens 的摘要
        summary_prompt = f"""请将以下剧情内容压缩为不超过150字的摘要，保留：
1. 关键事件和转折
2. 角色情感变化
3. 未解决的悬念和伏笔
4. 与下一集的衔接点

内容: {json.dumps(episode_full_content, ensure_ascii=False)}

输出纯文本摘要，不要JSON格式。"""
        return await self._llm_compress(summary_prompt)

    async def generate_arc_summary(self, arc_episodes: list) -> str:
        """自动生成故事弧摘要 — 弧结束时调用"""
        episode_summaries = [ep.summary for ep in arc_episodes]
        summary_prompt = f"""请将以下多个集的摘要合并为一个故事弧总结（不超过200字），保留：
1. 弧的核心冲突和解决
2. 角色发展轨迹
3. 对后续剧情的影响

各集摘要:
{chr(10).join(f'第{ep.number}集: {ep.summary}' for ep in arc_episodes)}

输出纯文本摘要。"""
        return await self._llm_compress(summary_prompt)

    async def update_project_summary(self, project_id: str) -> str:
        """更新项目级摘要 — 每 5 集自动触发"""
        arc_summaries = await self.get_all_arc_summaries(project_id)
        summary_prompt = f"""请将以下故事弧总结合并为项目整体概要（不超过300字），保留：
1. 主线故事脉络
2. 核心角色关系
3. 主要冲突和主题
4. 整体情感走向

各弧摘要:
{chr(10).join(arc_summaries)}

输出纯文本概要。"""
        return await self._llm_compress(summary_prompt)

5.5 批量生产管线中的摘要触发

class ProductionPipeline:
    """生产管线 — 含自动摘要机制"""

    async def produce_batch(self, project_id: str, start_episode: int, end_episode: int):
        """批量生产多集 — 自动管理上下文"""
        for ep_num in range(start_episode, end_episode + 1):
            # 1. 构建当前集上下文（自动压缩）
            context = await self.context_compressor.build_context_for_episode(
                project_id, ep_num, await self.get_project_settings(project_id)
            )

            # 2. 生成当前集
            episode = await self.produce_single_episode(project_id, ep_num, context)

            # 3. 生成集级摘要
            summary = await self.context_compressor.generate_episode_summary(episode)
            await self.save_episode_summary(project_id, ep_num, summary)

            # 4. 检查是否需要更新弧摘要
            if await self.is_arc_complete(project_id, ep_num):
                arc = await self.get_current_arc(project_id, ep_num)
                arc_summary = await self.context_compressor.generate_arc_summary(arc.episodes)
                await self.save_arc_summary(project_id, arc.id, arc_summary)

            # 5. 每 5 集更新项目级摘要
            if ep_num % 5 == 0:
                await self.context_compressor.update_project_summary(project_id)

class ImageStorageOptimizer:
    """图片存储优化器"""

    # 生成阶段直接指定目标分辨率，避免生成过大图片再缩放
    RESOLUTION_PRESETS = {
        "720p":  (1280, 720),
        "1080p": (1920, 1080),
        "4k":    (3840, 2160),
    }

    # 存储格式优先级
    STORAGE_FORMAT = "webp"            # WebP 比 PNG 小 60-70%，比 JPEG 小 25-35%
    WEBP_QUALITY = 85                  # 质量 85 在视觉无损和大小之间平衡

    # 中间态图片（如 AI 生成的原始输出）
    INTERMEDIATE_FORMAT = "webp"
    INTERMEDIATE_QUALITY = 80

    async def save_image(self, image_bytes: bytes, path: str, is_final: bool = False) -> str:
        """保存图片 — 自动压缩为 WebP"""
        quality = self.WEBP_QUALITY if is_final else self.INTERMEDIATE_QUALITY
        img = Image.open(io.BytesIO(image_bytes))

        # 自动裁剪到目标分辨率（如果超出）
        target_w, target_h = self.get_target_resolution()
        if img.width > target_w or img.height > target_h:
            img = img.resize((target_w, target_h), Image.LANCZOS)

        # 转换为 WebP
        output_path = path.rsplit(".", 1)[0] + ".webp"
        img.save(output_path, "WEBP", quality=quality, method=6)
        return output_path

    async def save_reference_image(self, image_bytes: bytes, character_id: str) -> str:
        """保存角色参考图 — 去重存储"""
        content_hash = hashlib.md5(image_bytes).hexdigest()
        existing = await self.find_by_hash(content_hash)
        if existing:
            # 已存在相同图片，只增加引用计数
            await self.increment_ref_count(existing.path)
            return existing.path
        return await self.save_image(image_bytes, f"refs/{character_id}/{content_hash}.webp")

class AudioStorageOptimizer:
    """音频存储优化器"""

    # 中间态音频使用 FLAC 无损压缩（比 WAV 小 50-60%，质量无损）
    INTERMEDIATE_FORMAT = "flac"

    # 最终输出使用 AAC（比 MP3 小 30% 且质量更好）
    FINAL_FORMAT = "aac"
    FINAL_BITRATE = "192k"

    # TTS 采样率：44.1kHz → 22.05kHz（语音场景足够）
    TTS_SAMPLE_RATE = 22050

    async def save_voice_audio(self, audio_bytes: bytes, path: str, is_final: bool = False) -> str:
        """保存配音音频"""
        if is_final:
            # 最终输出：AAC
            output_path = path.rsplit(".", 1)[0] + ".aac"
            audio = AudioSegment.from_file(io.BytesIO(audio_bytes))
            audio = audio.set_frame_rate(self.TTS_SAMPLE_RATE)
            audio.export(output_path, format="aac", bitrate=self.FINAL_BITRATE)
        else:
            # 中间态：FLAC 无损压缩
            output_path = path.rsplit(".", 1)[0] + ".flac"
            audio = AudioSegment.from_file(io.BytesIO(audio_bytes))
            audio = audio.set_frame_rate(self.TTS_SAMPLE_RATE)
            audio.export(output_path, format="flac")
        return output_path

class VideoStorageOptimizer:
    """视频存储优化器"""

    # 中间态视频编码设置
    INTERMEDIATE_CODEC = "libx265"     # H.265 比 H.264 小 40-50%

    INTERMEDIATE_PRESET = "medium"

    # 最终输出编码设置
    FINAL_CODEC = "libx265"
    FINAL_CRF = 23                     # CRF 23 高质量输出
    FINAL_PRESET = "slow"

    async def save_video_clip(self, video_path: str, is_final: bool = False) -> str:
        """保存视频片段 — 自动压缩"""
        crf = self.FINAL_CRF if is_final else self.INTERMEDIATE_CRF
        preset = self.FINAL_PRESET if is_final else self.INTERMEDIATE_PRESET
        output_path = video_path.rsplit(".", 1)[0] + "_compressed.mp4"

        ffmpeg.input(video_path).output(
            output_path,
            vcodec=self.FINAL_CODEC if is_final else self.INTERMEDIATE_CODEC,
            crf=crf,
            preset=preset,
            acodec="aac",
            audio_bitrate="128k",
        ).overwrite_output().run(quiet=True)

        return output_path

class StorageCleaner:
    """存储清理器 — 自动管理磁盘空间"""

    TEMP_FILE_MAX_AGE_HOURS = 24       # 临时文件 24 小时后自动清理
    INTERMEDIATE_KEEP_LATEST = 3       # 中间态文件只保留最新 3 个版本

    async def cleanup_temp_files(self):
        """定时任务：清理过期临时文件"""
        cutoff = datetime.now() - timedelta(hours=self.TEMP_FILE_MAX_AGE_HOURS)
        temp_dir = Path("./outputs/temp")
        for f in temp_dir.rglob("*"):
            if f.is_file() and datetime.fromtimestamp(f.stat().st_mtime) < cutoff:
                f.unlink()

    async def cleanup_old_intermediates(self, project_id: str):
        """清理旧版本中间态文件，只保留最新 N 个"""
        for asset_type in ["keyframes", "clips", "voices", "bgm"]:
            versions = await self.list_asset_versions(project_id, asset_type)
            if len(versions) > self.INTERMEDIATE_KEEP_LATEST:
                for old_version in versions[self.INTERMEDIATE_KEEP_LATEST:]:
                    await self.delete_asset(old_version)

        "sad":       {"speed": 0.85, "pitch": 0.9,  "energy": 0.7, "pause_ratio": 1.3},
        "angry":     {"speed": 1.2,  "pitch": 1.1,  "energy": 1.5, "pause_ratio": 0.6},
        "fearful":   {"speed": 1.15, "pitch": 1.2,  "energy": 0.9, "pause_ratio": 0.7},
        "surprised": {"speed": 1.1,  "pitch": 1.25, "energy": 1.3, "pause_ratio": 0.5},
        "calm":      {"speed": 0.95, "pitch": 1.0,  "energy": 0.8, "pause_ratio": 1.0},
        "whisper":   {"speed": 0.8,  "pitch": 0.85, "energy": 0.4, "pause_ratio": 1.5},
    }

    CHATTTS_EMOTION_TAGS: dict[str, str] = {
        "happy": "[laugh]", "sad": "[uv_break]", "angry": "",
        "fearful": "[uv_break]", "surprised": "", "calm": "", "whisper": "[uv_break]",
    }

    async def synthesize(
        self, dialogue: "Dialogue", character: "CharacterProfile"
    ) -> "AudioSegment":
        """合成带情感的角色配音 — 自动适配本地/第三方引擎"""
        params = self.EMOTION_PARAMS.get(dialogue.emotion, self.EMOTION_PARAMS["calm"])

        if self.tts_engine.engine_type == "chattts":
            emotion_tag = self.CHATTTS_EMOTION_TAGS.get(dialogue.emotion, "")
            text_with_emotion = f"{emotion_tag}{dialogue.text}"
            audio = await self.tts_engine.synthesize(
                text=text_with_emotion,
                spk_smp=character.voice_sample,
                temperature=params["energy"],
                top_P=0.7,
                top_K=20,
            )
        else:
            # CosyVoice / GPT-SoVITS / 第三方 TTS API — 统一参数接口
            audio = await self.tts_engine.synthesize(
                text=dialogue.text,
                speaker_id=character.voice_id,
                speed=params["speed"],
                pitch=params["pitch"],
                energy=params["energy"],
            )

        audio = self.adjust_volume(audio, dialogue.volume)

        if dialogue.pause_after_sec > 0:
            audio = audio + Silence(dialogue.pause_after_sec)

        return audio

class ComfyUIConfig:
    """ComfyUI 连接配置"""
    def __init__(self, base_url: str, ssl_verify: bool = True, timeout: int = 600) -> None:
        self.base_url = base_url.rstrip("/")
        self.ssl_verify = ssl_verify
        self.timeout = timeout
        self.scheme = "https" if base_url.startswith("https://") else "http"
        self.is_remote = not ("127.0.0.1" in base_url or "localhost" in base_url)

    def create_http_client(self) -> httpx.AsyncClient:
        transport = None
        if self.scheme == "https" and not self.ssl_verify:
            transport = httpx.AsyncHTTPTransport(verify=False)
        return httpx.AsyncClient(transport=transport, timeout=httpx.Timeout(self.timeout, connect=30.0))


class ComfyUIClientAdapter:
    """ComfyUI 客户端适配器 — 统一本地/远程/云调用"""
    def __init__(self, config: ComfyUIConfig) -> None:
        self.config = config
        self.client_id = str(uuid.uuid4())
        self._client: httpx.AsyncClient | None = None

    async def health_check(self) -> bool:
        client = await self._get_client()
        try:
            resp = await client.get(f"{self.config.base_url}/system_stats")
            return resp.status_code == 200
        except httpx.ConnectError:
            return False

14.6 版本兼容性检查脚本
 
python
python
#!/usr/bin/env python3
"""scripts/check_versions.py"""

import sys, subprocess, shutil

REQUIREMENTS = {
    "python":  {"cmd": None, "min": (3, 12, 0), "max": (3, 13, 0)},
    "node":    {"cmd": ["node", "--version"], "min": (22, 0, 0), "max": (23, 0, 0)},
    "pnpm":    {"cmd": ["pnpm", "--version"], "min": (9, 0, 0), "max": (10, 0, 0)},
    "ffmpeg":  {"cmd": ["ffmpeg", "-version"], "min": (7, 0, 0), "max": (8, 0, 0)},
    "psql":    {"cmd": ["psql", "--version"], "min": (17, 0, 0), "max": (18, 0, 0)},
    "redis":   {"cmd": ["redis-server", "--version"], "min": (7, 4, 0), "max": (8, 0, 0)},
}

def parse_version(raw: str) -> tuple[int, ...]:
    digits = []
    for part in raw.replace("v", "").split("."):
        try: digits.append(int("".join(c for c in part if c.isdigit())))
        except ValueError: break
    return tuple(digits[:3])

def check(name: str, spec: dict) -> bool:
    if spec["cmd"] is None: ver = sys.version_info[:3]
    else:
        if not shutil.which(spec["cmd"][0]):
            print(f"  [MISS] {name}: 未安装"); return False
        result = subprocess.run(spec["cmd"], capture_output=True, text=True)
        ver = parse_version(result.stdout + result.stderr)
    ok = spec["min"] <= ver < spec["max"]
    print(f"  [{'OK' if ok else 'FAIL'}] {name}: {'.'.join(map(str, ver))}  (需要 >= {'.'.join(map(str, spec['min']))})")
    return ok

def main() -> None:
    print("=== TVAiPlatform 版本兼容性检查 ===\n")
    results = [check(n, s) for n, s in REQUIREMENTS.items()]
    if all(results): print("\n✓ 所有版本检查通过")
    else: print("\n✗ 部分版本不满足要求"); sys.exit(1)

if __name__ == "__main__": main()

20.1 一致性评分

class ConsistencyScorer:
    def score(self, reference: "Image", generated: "Image") -> float:
        face_sim = self.face_similarity(reference, generated)
        body_sim = self.body_proportion_check(reference, generated)
        color_sim = self.color_consistency(reference, generated)
        return 0.5 * face_sim + 0.3 * body_sim + 0.2 * color_sim

二十一、关键技术难点与解决方案
 
难点	解决方案
角色跨年龄一致性	面部嵌入向量 + 年龄迁移LoRA + 固定骨骼结构

多服务统一适配	统一配置层 + 适配器模式，本地/远程/三方平台透明切换
字幕精确对齐	WhisperX 强制对齐 + 人工微调界面
无限集数支持	分层摘要(项目/弧/集) + 滑动窗口 + 上下文压缩引擎
JSON 模板驱动	模板工坊可视化编辑 + 版本管理 + 注入到 Dify 提示词
人工输入扩展性	全局设定 + 自定义维度 + 场景级覆盖，三层架构
存储空间优化	WebP/FLAC/H.265 自动压缩 + 参考图去重 + 临时文件自动清理
本地服务管理	Supervisor 进程管理 + 一键启停脚本 + 健康检查端点
数据一致性	PostgreSQL 17 事务保障 + Alembic 1.14 迁移版本控制
依赖版本冲突	版本锁定（pip + pnpm）+ 环境检查脚本 + mypy/ruff 静态分析
AI生成质量波动	每阶段人工审核节点 + 一致性评分自动重试 + 手工替换兜底
界面人性化	友好命名映射 + 操作提示 + 渐进式展示 + 引导式流程
